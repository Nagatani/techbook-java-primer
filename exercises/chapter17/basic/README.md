### 課題1: Hello World GUIアプリケーション
**ファイル名**: `HelloWorldGUI.java`, `HelloWorldGUITest.java`

基本的なGUIアプリケーションを作成し、Swingの基本を理解してください。

**要求仕様**:
- JFrameを使ったメインウィンドウ作成
- JLabel、JButton、JTextFieldの配置
- BorderLayoutによるレイアウト
- ボタンクリックイベントの処理
- ウィンドウの基本設定

**実行例**:
```
=== Hello World GUIアプリケーション ===
ウィンドウ表示: "Hello World GUI"
サイズ: 400x300ピクセル
配置: 画面中央

コンポーネント構成:
┌─────────────────────────────┐
│ タイトル: "Hello World GUI"     │ (上部)
├─────────────────────────────┤
│ 名前入力フィールド              │ (中央)
│ [田中太郎________________]      │
├─────────────────────────────┤
│ [挨拶する] [クリア] [終了]      │ (下部)
└─────────────────────────────┘

操作テスト:
1. 名前入力: "田中太郎"
2. [挨拶する]ボタンクリック
   → メッセージ表示: "こんにちは、田中太郎さん！"

3. [クリア]ボタンクリック
   → 入力フィールドがクリア

4. [終了]ボタンクリック
   → アプリケーション終了

ウィンドウイベント:
- 最小化/最大化: 対応
- サイズ変更: 対応
- 閉じるボタン: 確認ダイアログ表示

コンソール出力:
GUI初期化完了
名前入力: 田中太郎
挨拶実行: こんにちは、田中太郎さん！
入力フィールドクリア実行
アプリケーション終了
```

**評価ポイント**:
- 基本的なSwingコンポーネントの使用
- 適切なレイアウト設計
- イベント処理の実装

### 課題2: 簡単電卓アプリケーション
**ファイル名**: `SimpleCalculator.java`, `CalculatorTest.java`

四則演算ができる電卓GUIアプリケーションを作成してください。

**要求仕様**:
- 数字ボタン（0-9）と演算子ボタン（+、-、×、÷）
- 結果表示用のJTextField
- GridLayoutによるボタン配置
- 計算ロジックの実装
- エラーハンドリング（ゼロ除算等）

**実行例**:
```
=== 簡単電卓アプリケーション ===
ウィンドウ表示: "Simple Calculator"
サイズ: 300x400ピクセル

電卓レイアウト:
┌─────────────────────────────┐
│ [123.45________________]    │ (結果表示)
├─────────────────────────────┤
│ [C] [±] [%] [÷]             │
├─────────────────────────────┤
│ [7] [8] [9] [×]             │
├─────────────────────────────┤
│ [4] [5] [6] [-]             │
├─────────────────────────────┤
│ [1] [2] [3] [+]             │
├─────────────────────────────┤
│ [0] [.] [=]                 │
└─────────────────────────────┘

計算テスト:
操作: 1 + 2 =
表示: 1 → 1 + → 1 + 2 → 1 + 2 = 3

操作: 10 ÷ 3 =
表示: 10 ÷ 3 = 3.333333

操作: 15 × 4 =
表示: 15 × 4 = 60

操作: 100 - 25 =
表示: 100 - 25 = 75

エラーハンドリング:
操作: 10 ÷ 0 =
表示: "エラー: ゼロ除算"
[C]ボタンでリセット

連続計算:
操作: 5 + 3 = 8, + 2 = 10
表示: 前回の結果を引き継ぎ

特殊機能:
[C]ボタン: 全クリア
[±]ボタン: 正負反転
[%]ボタン: パーセント変換

履歴表示:
5 + 3 = 8
8 + 2 = 10
10 × 3 = 30
```

**評価ポイント**:
- GridLayoutの効果的な使用
- 計算ロジックの正確な実装
- ユーザビリティの考慮

### 課題3: テキストエディタアプリケーション
**ファイル名**: `SimpleTextEditor.java`, `TextEditorTest.java`

基本的なテキストエディタGUIアプリケーションを作成してください。

**要求仕様**:
- JTextAreaによるテキスト編集エリア
- JScrollPaneによるスクロール機能
- メニューバー（ファイル、編集、表示）
- ファイルの開く・保存機能
- 検索・置換機能

**実行例**:
```
=== テキストエディタアプリケーション ===
ウィンドウ表示: "Simple Text Editor"
サイズ: 600x400ピクセル

メニュー構成:
ファイル(F)
├── 新規作成 (Ctrl+N)
├── 開く (Ctrl+O)
├── 保存 (Ctrl+S)
├── 名前を付けて保存
├── ──────────
└── 終了 (Ctrl+Q)

編集(E)
├── 元に戻す (Ctrl+Z)
├── やり直し (Ctrl+Y)
├── ──────────
├── 切り取り (Ctrl+X)
├── コピー (Ctrl+C)
├── 貼り付け (Ctrl+V)
├── ──────────
├── 検索 (Ctrl+F)
└── 置換 (Ctrl+H)

表示(V)
├── フォント設定
├── 文字コード
└── ステータスバー

エディタ画面:
┌─────────────────────────────┐
│ [メニューバー]              │
├─────────────────────────────┤
│ ┌─スクロール可能テキストエリア─┐ │
│ │ Hello, World!           │ │
│ │ Java Swing Text Editor  │ │
│ │ 複数行のテキスト編集が    │ │
│ │ 可能です。              │ │
│ │ ■                      │ │ (カーソル)
│ └───────────────────────────┘ │
├─────────────────────────────┤
│ 行: 5, 列: 1 | 文字数: 67    │ (ステータスバー)
└─────────────────────────────┘

ファイル操作テスト:
1. [新規作成]: 空のドキュメント
2. テキスト入力: "Hello, World!"
3. [保存]: sample.txt として保存
4. [開く]: sample.txt を読み込み

検索機能テスト:
検索文字列: "Hello"
結果: 1箇所見つかりました（ハイライト表示）

置換機能テスト:
検索: "Hello"
置換: "Hi"
結果: "Hi, World!"

フォント設定:
フォント: MS Gothic
サイズ: 14pt
スタイル: 標準

ステータス情報:
現在行: 5
現在列: 1
総文字数: 67文字
総行数: 5行
文字コード: UTF-8
```

**評価ポイント**:
- メニューシステムの実装
- ファイルI/Oとの連携
- ユーザビリティ機能の実装

### 課題4: 個人情報管理アプリケーション
**ファイル名**: `ContactManager.java`, `Contact.java`, `ContactManagerTest.java`

連絡先を管理するGUIアプリケーションを作成してください。

**要求仕様**:
- JTableによるデータ表示
- フォーム入力による新規登録・編集
- 検索・フィルタリング機能
- データの永続化（ファイル保存）
- TableModelの実装

**実行例**:
```
=== 個人情報管理アプリケーション ===
ウィンドウ表示: "Contact Manager"
サイズ: 800x600ピクセル

メイン画面レイアウト:
┌─────────────────────────────┐
│ [新規] [編集] [削除] [検索]   │ (ツールバー)
├─────────────────────────────┤
│ ┌─連絡先一覧テーブル───────────┐ │
│ │ ID │ 名前    │電話番号   │ │
│ │ 1  │田中太郎 │090-1234-5678│ │
│ │ 2  │佐藤花子 │080-9876-5432│ │
│ │ 3  │鈴木一郎 │070-1111-2222│ │
│ └───────────────────────────┘ │
├─────────────────────────────┤
│ 詳細情報:                   │
│ 名前: 田中太郎               │
│ 電話: 090-1234-5678         │
│ メール: tanaka@example.com  │
│ 住所: 東京都渋谷区...       │
└─────────────────────────────┘

新規登録フォーム:
┌─────────────────────────────┐
│ 連絡先の新規登録             │
├─────────────────────────────┤
│ 名前: [________________]    │
│ 電話: [________________]    │
│ メール: [______________]    │
│ 住所: [________________]    │
│      [________________]    │
│ 会社: [________________]    │
│ 部署: [________________]    │
├─────────────────────────────┤
│      [保存] [キャンセル]     │
└─────────────────────────────┘

操作テスト:
1. [新規]ボタンクリック
   → 入力フォーム表示

2. 情報入力:
   名前: 山田次郎
   電話: 090-5555-6666
   メール: yamada@example.com

3. [保存]ボタンクリック
   → テーブルに新しい行が追加

4. テーブル行選択
   → 詳細情報表示

5. [編集]ボタンクリック
   → 選択した連絡先の編集フォーム

6. [削除]ボタンクリック
   → 確認ダイアログ後削除

検索機能:
検索フィールド: "田中"
結果: 田中太郎の行のみ表示

データ管理:
保存ファイル: contacts.json
形式: JSON形式でのデータ永続化
自動保存: 編集後に自動でファイル保存

統計情報:
総連絡先数: 15件
最近追加: 山田次郎 (2024-07-04)
最近更新: 佐藤花子 (2024-07-03)

データバックアップ:
バックアップファイル: contacts_backup_20240704.json
自動バックアップ: 毎日実行
```

**評価ポイント**:
- JTableとTableModelの活用
- CRUD操作の実装
- データ永続化の実装

## 🔍 GUI基礎のポイント

1. **コンポーネント階層**: Container → Component の関係
2. **レイアウトマネージャー**: BorderLayout、GridLayout、FlowLayout
3. **イベント処理**: ActionListener、MouseListener等
4. **スレッド**: Event Dispatch Thread（EDT）での実行
5. **Look & Feel**: プラットフォーム固有の外観
6. **MVC**: Model-View-Controller パターンの理解
